gitbook的使用

1. 安装gitbook

mac下安装gitbook

2. 目录结构

gitbook项目的目录结构如下:

- SUMMARY.md
- README.md
- book.json
- _book/
- node_modules/

其中，我们需要首先创建的是SUMMARY.md(书籍目录) 和 README.md(书籍介绍)

3. SUMMARY.md

书籍目录是gitbook的基础，可以有层级结构，使用markdown写法

# Summary

* [git相关](git.md)
    * [I. 为不同的git项目配置各自独立的用户](./git/user.md)
    * [II. gitbook的使用](./git/gitbook.md)
    ...

4. 编译和预览书籍

书籍目录和目录中链接的内容(*.md)文件编辑完成后，执行gitbook serve即可完成书籍的编译

默认的"Introduction"页面就是README.md

5.2. 载入配置

  • 在项目目录下执行gitbook install即可安装配置插件；安装完成后，项目下会出现一个node_modules/目录
  • 然后执行gitbook serve即可看到配置完成后的主页

6. 集成发布到github pages

6.1. 添加git仓库

  1. 在项目目录下执行git init新建git仓库
  2. git remote add origin <远端github仓库地址> 添加github仓库

  3. touch .gitignore 并且在里面添加以下内容

     *~
 *.swp
 *.DS_Store
 _book/
 node_modules/

  4. git add . 添加文件到git版本控制

  5. git commit -m 'git init' 提交变动
  6. git push origin master 推送分支

6.2. 发布github pages

  1. git clone <远端仓库地址> <目的目录>
  2. 在<目的目录>下执行git checkout --orphan gh-pages 创建孤儿分支gh-pages
  3. git rm --cached -r . 删除暂存区目录
  4. git clean -df 删除未track的目录，不影响.gitignore中的文件

  5. touch .gitignore 并且在里面添加以下内容

     *~
 *.swp
 *.DS_Store

  6. cp -rf <源目录>/_book/* ./ 将生成的书籍html内容copy出来

  7. git add .
  8. git commit -m 'publish book'
  9. git push origin gh-pages

  10. 推送完成后，进入github仓库主页，会多出一个environment图标

  11. 点击environment图标后，进入如下页面

  12. 点击view deployment图标，即可看到发布的gitbook书籍

6.3. 后续更新github pages

  • 只需执行上述6 - 9 步即可
